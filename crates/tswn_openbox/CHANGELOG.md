# 更新日志

## [0.2.0] - 2026-05-26

### 新增

- `to-diy` 新增可选输出文件，支持像 CLI 的 `-o/--out-file` 一样直接把导出结果写入文件。
- `namer-pf` 新增可选输出文件，便于批量评分结果直接落盘。
- `batch-rate` 与 `pair` 不再强制要求输出文件；未勾选“写入文件”时会只输出到右侧日志面板。
- `batch-rate` 与 `pair` 新增 `perf` 开关，日志中可额外输出 total/init/fight 的耗时拆分，便于对齐 CLI 的 `--perf`。
- 输入文件模式新增“刷新预览”按钮，方便在外部修改文件后直接复读当前输入。

### 重构

- 将原先堆在 `main.rs` 的 GUI 状态、输入源、控件与任务启动逻辑拆到 `src/app/`。
- 将原先堆在 `backend.rs` 的解析、评分、格式化与执行逻辑拆到 `src/backend/`。
- 统一了可选输出文件与日志追加逻辑，减少各工具各自维护一套输出分支。

### Windows

- Windows 构建启用 `windows_subsystem = "windows"`，双击启动 GUI 时不再额外挂出控制台窗口。

### 修复

- `batch-rate` 的 `日志阈值` / `文件阈值` 现在明确校验为 `0~100`。
- `pair` 的阈值现在明确校验为非负数，避免非法输入被静默当作空值处理。
- 日志追加逻辑改为统一收口，避免任务完成消息和多行输出之间出现重复空行。
