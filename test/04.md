## 关键现象

1. 这是早中段分叉（前 7 条以内）。
2. 分叉点正好落在“应当触发幻术召唤”处，实际变成了另一名角色的攻击行动。
3. 该偏移会快速改写后续召唤物/背刺/反击链，导致 trace 长度明显偏移。

## 原因判断（按置信度）

### 高置信
- 这类“应召唤却未召唤”的分叉，和 case02 一样，最像技能选择阶段随机消费/目标语义不一致引起的行动序漂移。

### 中高置信
- 本例中早期已有使魔（血祭）和潜行（背刺前置），spawn 进入队列后的 `alives`/`players` 插入位置若与 JS 有细微差别，会放大分叉：
  - Rust `add_new_player`：`runners.rs:89-111`
  - Rust 同步时机：`runners.rs:341-355` 与 `395`

### 中置信
- `pick_skip_range` 与 JS `pickSkipRange` 的参数语义不同步风险，在“多阵营 + 召唤物插入”场景下更容易触发。
