## 关键现象

1. 这是**超早分叉**（第 4 条事件）。
2. 分叉发生在“召唤幻影”之后立即出现，和“召唤后 RC4 流漂移”高度一致。
3. 偏离表现是“行动者顺序变化”，不是单纯伤害数字偏差。

## Rust 侧关键链路（参考）

- 行动主流程：`src/player/impl_runtime.rs:29-87`
- 统一目标选择逻辑：`src/player/impl_runtime.rs:140-189`
- 统一目标评分会在非 smart 下消费 `rFFFF`：`src/player/skill.rs:190-192`
- Shadow 技能（Rust）：
  - `target_domain=SelfOnly`、`select_target_count=1`：`src/player/skill/act/shadow.rs:28-31`
  - `select_targets_with_level` 返回自身：`.../shadow.rs:44-46`
- 但 runtime 当前**没有走 skill.select_targets**（而是走统一 `select_skill_targets`），对应入口：`impl_runtime.rs:57,72`

## JS 侧对照（参考）

- `Plr.action`：`fast-namerena/branch/latest/md5.js:18249-18275`
- `SklShadow.aa` 直接返回空数组（无目标采样/打分随机）：`.../md5.js:15673-15675`
- 通用 `Skill.aa`（会采样 + score）：`.../md5.js:18618-18647`

## 根因判断（按置信度）

### 高置信（本用例最关键）
- **幻术目标选择路径的随机消费与 JS 不一致**。  
  JS 的 `SklShadow.aa` 返回 `[]`，几乎不引入目标评分随机；Rust runtime 的统一目标选择会走评分分支，非 smart 会调用 `rFFFF`，导致额外 RC4 消费，刚好能解释“idx=2 后立刻在 idx=3 分叉”。

### 中置信
- `pick_enemy_target -> pick_skip_range` 的实现语义（按索引批量跳过）与 JS `pickSkipRange`（按 first + skip_len）在复杂队列下可能进一步放大偏差：
  - Rust：`src/player/impl_runtime.rs:103-119`, `src/rc4.rs:497-506`
  - JS：`md5.js:19867-19877`

## 建议验证顺序

1. 专盯 `idx=2` 前后，对 `Sayaka` 的幻术一次行动做逐调用 RC4 对账。
2. 核对“是否走到 shadow 的技能自定义选目标语义”，避免 runtime 统一逻辑额外消费随机。
3. 该点对齐后再回归 case01/03/04，确认是否连带收敛。
