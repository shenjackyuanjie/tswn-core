# tswn_core 0.2.3 更新说明（`--out-raw` 聚合日志输出）

> 这版主要补齐 CLI 的一条新输出通道：让 Rust 侧可以直接吐出更接近 `fast-namerena` / `md5.js` 对账习惯的 raw 聚合战斗日志，方便做日志级比较与回归分析。
>
> 对应简版条目见：`crates\tswn_core\CHANGELOG.md`

---

## 1. 改动目标

在 `0.2.2` 完成版本标识与 CLI 启动文案整理后，`0.2.3` 的重点回到**输出格式能力**。

当时的需求很明确：

- Rust CLI 需要一种更适合直接和基准实现对账的日志模式；
- 输出不应只是逐条 `RunUpdate` 原样打印，而要更接近现有工具链使用的聚合格式；
- 新模式不能影响已有普通输出、图标命令和 benchmark 命令的行为。

所以这版的核心不是“新增战斗机制”，而是“新增一条更适合分析的日志视图”。

---

## 2. 主要改动

## 2.1 新增 `--out-raw` 命令行开关

涉及文件：`crates\tswn_core\src\bin\tswn_cli.rs`

CLI 新增：

- `--out-raw`

这个开关只在**普通对战模式**下生效，用来输出聚合后的 raw 战斗日志。

### 设计点

实现上先把参数解析拆成两层：

- 一层专门收集真正的业务参数；
- 一层单独识别 `--out-raw` 这种“输出模式 flag”。

这样可以保证：

- `--help`
- `--icon`
- `--icon-b64`
- `--icon-path`
- `--bench` / `--win_rate` 等 benchmark 命令

都不会被这个 flag 误伤。

---

## 2.2 输出格式改为“按动作聚合”

涉及文件：`crates\tswn_core\src\bin\tswn_cli.rs`

为了对齐 `fast-namerena/branch/latest/out_md5.ts` 一类工具的习惯，本版没有直接打印每条原始 update，而是新增了一整套聚合输出流程：

- 先对单条日志做格式化；
- 过滤控制字符和零宽字符；
- 归一化多余空白；
- 识别“动作起始行”；
- 把同一动作后续产生的事件拼接到同一段输出里；
- 用空行分隔不同动作/段落。

### 结果

启用 `--out-raw` 后，输出特征变成：

- 按回合/动作分段；
- 一次行动后的后续事件会继续挂在该动作行后面；
- 用空行分隔段落；
- 更适合直接做文本 diff 或与 JS/Dart 输出肉眼比较。

这让 Rust CLI 从“能打印日志”进一步变成“能打印适合对账的日志”。

---

## 2.3 Raw 模式下关闭额外装饰输出

涉及文件：`crates\tswn_core\src\bin\tswn_cli.rs`

在 `--out-raw` 模式下，CLI 会主动关闭以下常规输出：

- 欢迎信息；
- 玩家初始状态列表；
- 对局结束后的结果摘要。

### 原因

如果这些附加信息仍然保留，日志 diff 时会混入很多与战斗过程无关的文本，反而降低 raw 模式的价值。

所以 `0.2.3` 的选择很明确：

- 普通模式继续保留完整的人类可读输出；
- `--out-raw` 模式只服务于“战斗过程文本对账”。

---

## 2.4 CLI 帮助与示例同步补齐

涉及文件：`crates\tswn_core\src\bin\tswn_cli.rs`

本版也同步更新了 CLI 的帮助文本和示例：

- `--help` 里加入 `--out-raw` 说明；
- 示例命令增加 raw 输出模式的用法；
- 输入读取逻辑也做了相应整理，保证 flag 解析和原始输入解析不互相干扰。

这让新能力不只“代码里能用”，而是“从帮助信息就能发现并直接试用”。

---

## 3. 影响 / 对齐点 / 兼容性

这版**不改变核心战斗语义**，主要影响 CLI 输出层。

具体来说：

- 默认输出行为保持原样；
- 只有显式传入 `--out-raw` 时，才切换到聚合日志模式；
- benchmark、图标生成和帮助命令不会被这个开关改写行为。

对外可见的变化主要是：Rust CLI 现在可以更直接地生成适合和基准实现对账的文本输出。
