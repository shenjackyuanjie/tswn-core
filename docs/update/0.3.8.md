# tswn_core 0.3.8 更新说明

`0.3.8` 是一次以 **DIY / Overlay 收口** 和 **核心性能优化** 为主的版本。相比 `0.3.7`，这一版没有刻意扩展新的玩法规则，而是集中处理上一轮 DIY 能力增强之后暴露出来的结构问题、clone 继承细节、导出格式细节，以及胜率 / benchmark 路径上的热区开销。

这次优化量比较大，尤其是固定 30-case 性能集合下，已经能看到明显收益。后续如果继续做引擎结构调整，建议以本版本新增和整理后的固定样本、分组汇总和 track 工具作为主要性能回归口径。

## 版本定位

- 当前版本：`tswn_core 0.3.8`
- 上一版本：`tswn_core 0.3.7`
- 重点方向：
  - DIY / Overlay 表达和构造路径收口；
  - DIY clone、技能加成、技能顺序相关 bug 修复；
  - `to-diy` 导出结果精简；
  - 引擎热路径和 benchmark runner 大幅降本；
  - 固定性能样本扩展到 30 个，并新增分组汇总。

## DIY / Overlay

### 技能覆盖结构改为有序槽位

本轮将 DIY / Overlay 的技能覆盖结构从“按名称映射 + 额外顺序字段”的组合，调整为更直接的有序技能槽表达。

此前的结构需要同时维护技能映射和 `skill_order`，在构造、导出、clone 重建等路径上都要额外保证两份数据一致。`0.3.8` 改为使用保持顺序的技能槽列表后，技能顺序本身由数据结构表达，减少了重复状态，也降低了后续维护难度。

带来的影响包括：

- DIY 技能覆盖的顺序语义更直接，不再依赖额外的 `skill_order` 字段；
- `to-diy` / `to_ol_json` 这类导出路径可以直接遍历有效技能槽；
- clone 重建时更容易复用 owner 的 DIY 技能配置；
- 直接依赖 `PlayerOverlay.skills` 具体类型的调用方需要按新的 ordered skill slots 结构适配。

### 修复 LastBoost / SlotBoost 应用顺序

`0.3.7` 引入 `SkillBoost` 后，DIY 技能需要同时表达：

- 普通等级：`Normal(lv)`；
- 末位座位加成：`SlotBoost { base, boost }`；
- 末位主动技翻倍：`LastBoost(base)`。

`0.3.8` 修复了普通构造和 DIY 构造中 `LastBoost` / `SlotBoost` 的应用顺序问题。现在基础等级覆盖、末位加成、末位翻倍之间的关系更清晰，避免出现普通 build 与 DIY build 在相同输入下计算顺序不一致的问题。

这部分主要影响：

- 带末位技能加成的 DIY 名字；
- 从普通名字导出后再回灌的 DIY / OL 配置；
- 涉及技能衰减下限的 clone 路径。

### 修复 DIY clone 重复加成

上一版已经对 DIY clone 的衰减下限做了处理，但在部分路径下，clone 技能重建会把 overlay 加成和衰减下限逻辑叠加得过多，导致技能等级被二次放大。

本版本将普通构造和 DIY clone 的 boost 路径进一步拆开：

- 普通构造按正常名字、座位、末位规则应用技能加成；
- DIY 构造以 overlay 中记录的 `base_level` / `SkillBoost` 为准；
- clone 重建时复用 DIY 元数据，但避免重复执行已经体现在 overlay 中的加成。

这个修复能让 DIY 分身后的技能等级更符合“以 owner 当前技能与 DIY 规则为基准，再按 clone 规则衰减”的预期。

### `to-diy` 输出更紧凑

`to-diy` 导出路径现在会跳过 0 级技能，并修复因此产生的多余逗号问题。

之前在部分名字导出为 DIY / OL 时，0 级技能可能仍然出现在输出里，或者跳过后留下类似多余分隔符的格式瑕疵。现在导出结果更接近“实际有效配置”：

- 0 级技能不会污染导出；
- 多余逗号被移除；
- 字符串构造更少，导出结果也更短；
- 对 roundtrip 验证更友好。

### 文档同步

`docs/DIY.md` 已同步更新，重点补充了：

- skills-as-ordered-Vec / ordered skill slots 的当前表达；
- DIY clone 行为和技能继承细节；
- 验证状态与 roundtrip 相关说明；
- `to-diy` 输出行为变化。

## 性能优化

`0.3.8` 的另一条主线是性能优化。本轮不是单点 micro optimization，而是围绕胜率计算、固定样本 benchmark、回合推进和状态 / hook 热路径做了一轮比较系统的降本。

### 引擎热路径优化

本轮优化覆盖了多个核心模块：

- `engine_core` / `tick` / `runners`：减少回合推进中的不必要分支和重复访问；
- `world_state` / `storage`：降低玩家状态读取、存储访问和 alive 信息维护的开销；
- `impl_runtime` / `skill store`：减少技能运行时路径中的重复计算；
- 多个主动技 / 被动技文件：配合新路径调整调用和状态处理方式。

整体目标是减少每场对局中反复执行的基础成本，尤其是 1v1、2v2 这种核心胜率场景里“每场很短但跑很多场”的路径。

### 跳过空 hook / 空状态路径

本轮明确优化了几个以前在热路径中会反复经过、但大多数情况下没有实际工作的路径：

- 空 `pre-step` 路径；
- 空 action 状态 hook；
- 空 defend hook；
- 空 damage hook。

这些 hook 对功能扩展和技能系统很重要，但对于大量普通对局来说，很多回合并没有需要执行的对应回调。`0.3.8` 在保留语义的前提下，让这些“没有工作要做”的路径尽早跳过，减少无效迭代和函数调用成本。

这类优化对单场复杂逻辑不一定显眼，但在 `win_rate` / benchmark 这种几十万、几百万场累计执行的场景下收益非常明显。

### DIY 相关性能优化

DIY 路径本身也做了性能收口：

- 技能槽改为有序结构后，减少了顺序维护成本；
- 对重复 ordered skill slots 做去重，避免重复技能槽进入后续构造和导出路径；
- `to-diy` 跳过 0 级技能，减少无意义字符串拼接；
- 导出字符串更短，roundtrip / 对比工具中的额外开销也随之下降。

这些优化主要影响 DIY 构造、导出、验证工具和批量 roundtrip 场景；对普通非 DIY 对战路径也尽量保持低开销。

### Benchmark runner 降本

之前固定样本 benchmark 中，runner 本身也会贡献一部分基础开销。`0.3.8` 降低了 benchmark runner 的额外成本，使采样结果更接近核心引擎真实消耗。

同时 release profile 调整为单 codegen unit，便于在本地长期跟踪时获得更稳定的 release benchmark 结果。

## 固定 30-case 性能口径

本轮把固定性能样本从 20 个扩展到了 30 个，并把它整理成长期维护的固定性能回归集合：

- 路径：`docs/perf/fixed_cases_30/*.txt`
- 工具：`track_perf_cases`
- 推荐参数：`--release --features aux_bins,no_debug`
- 默认单线程：`--thread 1`
- 每个 case：`13000` 场
- 总量：`30 * 13000 = 390000` 场

新增的 `21` 到 `30` 号样本主要强化 1v1 / 2v2 核心路径，包括短局爆发、常规 1v1、长回合消耗、高频技能触发、高频 buff/debuff、随机 / 目标选择、极端边界、2v2 群体技能和队友交互等场景。

### 分组汇总

`track_perf_cases` 现在会额外输出分组汇总，方便快速判断是哪类路径出现变化：

- `overall`：全部 30 个 case；
- `core_1v1_2v2`：核心 1v1 / 2v2 样本；
- `one_v_one`：只看 1v1；
- `two_v_two`：只看 2v2；
- `stress_multi`：多人压力样本，如 FFA、3v3v3 等。

这个分组很重要：如果只看 `overall`，多人重型 case 容易掩盖 1v1 / 2v2 的主路径变化；反过来，如果只看 1v1，也可能漏掉多人状态链和召唤 / 复活路径的退化。

### 当前记录数据

当前固定 30-case 记录中，最新一次结果为：

| 分组           | case 数 |   runs |   elapsed |   us/场 |    场/s | init us/场 | fight us/场 |
| -------------- | ------: | -----: | --------: | ------: | ------: | ---------: | ----------: |
| `overall`      |      30 | 390000 | `29.905s` |  `76.7` | `13041` |      `9.2` |      `63.4` |
| `core_1v1_2v2` |      17 | 221000 |  `8.738s` |  `39.5` | `25292` |      `5.7` |      `31.3` |
| `one_v_one`    |      11 | 143000 |  `3.654s` |  `25.6` | `39138` |      `4.5` |      `19.3` |
| `two_v_two`    |       6 |  78000 |  `5.084s` |  `65.2` | `15342` |      `8.1` |      `53.3` |
| `stress_multi` |       8 | 104000 | `16.206s` | `155.8` |  `6417` |     `17.0` |     `131.6` |

作为对照，`docs/perf/fixed_cases_30_benchmark.md` 中记录的同机目标口径为 `0.3.7` / 单线程 / `--features no_debug` 下约 `40.507s`。本轮优化后的固定集合结果约为 `29.905s`，说明这轮热路径回收幅度很可观。

后续如果只做本地快速试跑，可以使用 `--profile release-fast`；正式写入长期性能记录时仍使用 `--release`。benchmark 口径不启用 `mimalloc_alloc`。

具体运行命令：

```powershell
cargo run --release --features aux_bins,no_debug --bin track_perf_cases -- `
  --case-dir docs/perf/fixed_cases_30 `
  --out-dir docs/perf/fixed_cases_30_results `
  --bench-runs 13000 `
  --thread 1
```

## 跟踪与验证工具

本轮新增 / 扩展了多组 Rust 侧跟踪工具，方便后续继续定位性能和一致性问题：

- `track_perf_cases`：固定样本性能采样、分组汇总、版本化输出；
- `track_diy_roundtrip`：DIY / OL 导出回灌验证；
- `track_case_miner`：从号库生成和筛选候选 case；
- `track_test` / `track`：辅助定位指定 case 的运行行为；
- `case_gen`：抽取公共 case 生成逻辑，供 miner 和 perf 工具复用。

这些工具的意义不只是临时排查，而是让之后的性能优化和 DIY 行为修复有稳定的复现口径：

- 行为修复可以先跑 roundtrip / fixed case；
- 性能优化可以看 30-case 分组；
- 新增复杂机制时可以补充固定样本，避免只靠随机 case 观察。

## 行为与兼容性

### 对普通对战的影响

本版本主要改动集中在 DIY 和热路径优化。普通非 DIY 对战的规则语义没有刻意改变；性能优化也以保留现有结果为前提。

不过，由于引擎热路径、状态访问和 hook 路径有较多内部调整，如果后续发现 replay / diff 个别 case 有变化，建议优先用固定 case 和 track 工具缩小范围。

### 对 DIY 调用方的影响

如果调用方只是通过名字字符串、`diy[...]` 或 `ol:{...}` 使用 DIY 功能，迁移成本应较低，主要感知是：

- 输出更短；
- 0 级技能不再出现；
- clone 后技能加成更稳定；
- 末位技能 boost 顺序更符合预期。

如果调用方直接构造或读取 `PlayerOverlay.skills`，则需要注意内部结构已经从旧的映射 + 顺序字段，调整为 ordered skill slots。此类代码需要按新结构适配。

### 对文档和性能记录的影响

本版本之后，建议把 `docs/perf/fixed_cases_30_benchmark.md` 作为主要性能回归入口；旧的两个样本表仍可用于历史横向比较，但覆盖面已经不够判断当前引擎复杂路径。

## 总结

`0.3.8` 可以视为 `0.3.7` 后的一次“收口 + 加速”版本：

- DIY 技能结构更清晰；
- clone / boost / 导出路径更稳定；
- `to-diy` 输出更干净；
- 引擎热路径明显降本；
- benchmark 体系更适合长期维护。

如果只关心普通胜率计算，本版本最明显的收益是速度；如果使用 DIY / Overlay，本版本同时修复了若干会影响导出、clone 和末位技能加成的细节问题。
