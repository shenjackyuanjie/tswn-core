# tswn_core 0.2.15 更新说明（新增 batch-rate 批量胜率命令、补齐中文注释）

> `0.2.15` 这一版主要围绕 `tswn-cli` 的批量胜率评估体验做补强：一方面补上面向文件输入的 `bench batch-rate`，另一方面把这轮新增命令里的注释和帮助文案统一整理成中文，并顺手修掉一个新命令参数透传顺序的问题，同时补上适合长批次任务消费的逐行文件输出能力。
>
> 这轮改动主要有五件事：
>
> 1. 新增 `tswn-cli bench batch-rate`（别名 `cqp`）批量胜率命令；
> 2. 支持从靶子列表和选手列表文件批量读取 `+` 分组输入，并保留原始行文本作为输出标签；
> 3. 支持通过 `--out-file/-o` 将批量结果按 JSONL 一行一个结果写入文件，并在每行后立刻 flush；
> 4. 为一批常用 CLI 参数补齐短版别名，减少命令输入长度；
> 5. 将相关注释统一改成中文，并修正 `--perf` / `--verbose` 的执行透传顺序。
>
> 对应简版条目见：`crates\tswn_core\CHANGELOG.md`

---

## 1. 改动目标

这次不是继续扩核心战斗逻辑，而是补一条更适合批量评估的 CLI 使用路径。

在已有的 `bench win-rate` 和 `bench group-win-rate` 基础上，CLI 已经能处理单次双队或“一个目标组对多个对手组”的场景，但如果要做更大批量的筛选，例如：

- 一份固定靶子池
- 一份待筛选选手池
- 希望直接看每个选手组对整个靶子池的平均胜率

原来的命令组合就不够顺手，通常需要外层脚本自己循环拼接输入。

因此，`0.2.15` 的目标很明确：

- 让 CLI 直接支持“从两个列表文件批量跑平均胜率”；
- 保持和现有 `bench` 子命令一致的 `-n`、`-t`、`--single-thread`、`--perf`、`--keep-rq` 语义；
- 把本轮新增代码里的英文注释统一回中文，避免后续继续维护时中英混杂。

## 2. 主要改动

## 2.1 新增 `bench batch-rate` / `bench cqp`

涉及文件：

- `crates\tswn_core\src\bin\tswn_cli.rs`
- `crates\tswn_core\src\bin\tswn_cli\args.rs`
- `crates\tswn_core\src\bin\tswn_cli\bench.rs`

这一版新增了：

- `tswn-cli bench batch-rate`
- 可见别名 `tswn-cli bench cqp`

它的输入形式是两份文件：

- `--target-list <FILE>`：靶子组列表
- `--player-list <FILE>`：选手组列表

两份文件都遵循同一格式：

- 每个非空行代表一组
- 组内成员使用 `+` 分隔
- 会自动跳过空行

例如：

```text
mario
peach+daisy
bowser+koopa
```

其中第二、三行会分别被解析成多人成组输入，并在内部转换成 namerena 需要的换行分组格式。

---

## 2.2 批量输出会保留原始行标签

为了让批量结果更适合直接阅读，这次解析选手列表时，不只保留转换后的组字符串，还会额外保留文件中的原始行文本。

这意味着默认输出会像这样按原始行标签展示：

- `mario`
- `peach+daisy`
- `bowser+koopa`

这样在批量跑完后，可以直接把每个选手组对应回源文件里的配置文本，而不需要再把内部的换行格式重新反解出来。

同时：

- 默认输出每个选手组的平均胜率与吞吐信息；
- 追加 `--verbose` 后，会逐个靶子输出明细胜率；
- 追加 `--perf` 后，会继续打印 `total/init/fight` 的计时拆分。

如果追加：

- `--out-file <FILE>`，则会把每个选手组的汇总结果按 JSONL 写入文件；
- 每个选手组只写一行，便于长批次任务边跑边消费；
- 每写完一行都会立即 `flush()`，避免长时间运行时结果只停留在缓冲区里；
- 如果目标文件已存在，则会先询问覆盖；若不覆盖，再询问是否追加；也可以用 `--force/-f` 直接覆盖。

这一版还顺手给常用参数补齐了短版，主要包括：

- `--raw/-r`
- `--file/-f`
- `--single-thread/-s`
- `--target/-l`
- `--target-list/-l`
- `--player-list/-p`

## 2.3 统一整理为中文注释与帮助文案

这轮改动里新增的 `batch-rate` 相关代码原本还残留了一些英文注释。

`0.2.15` 中已经同步整理为中文，并补齐了原本缺失的参数说明，主要包括：

- `ParsedCommand::BenchBatchRate` 内部字段注释
- `BenchBatchRateCommand` 的参数说明
- `+` 分组解析辅助函数注释
- CLI 顶层模块说明里的子命令列表和示例

这样后续继续维护 CLI 时，阅读体验会更统一，也更符合当前仓库整体的中文文档风格。

---

## 2.4 修正 `--perf` / `--verbose` 透传顺序

在本轮新增命令接线时，`bench batch-rate` 调用执行函数的参数顺序里，`--perf` 和 `--verbose` 一度被传反。

这个问题不会导致编译失败，但会造成运行时行为对调：

- 开了 `--perf` 却表现成明细输出；
- 开了 `--verbose` 却去打印性能统计。

这次已经一并修正，确保两个开关重新按字面语义生效。

## 3. 使用示例

示例命令：

```bash
tswn-cli bench batch-rate --target-list targets.txt --player-list players.txt -n 10000 -t 8
tswn-cli bench cqp --target-list targets.txt --player-list players.txt -n 10000 --verbose --perf
tswn-cli bench batch-rate --target-list targets.txt --player-list players.txt -n 10000 --keep-rq
tswn-cli bench batch-rate --target-list targets.txt --player-list players.txt -o result.jsonl --force
```

适用场景包括：

- 固定一批靶子，对候选选手组做快速筛选；
- 用 `--verbose` 查看某个选手组究竟是被哪几个靶子压低了均值；
- 用 `--keep-rq` 对照普通 `fight + seed:i@!` 风格下的结果口径。
