# tswn_core 0.3.2 更新说明（DIY / overlay 覆盖系统）

> `0.3.2` 新增 `PlayerOverlay` 覆盖系统，允许在名字后缀中直接指定玩家的八围属性、技能等级和武器名，跳过正常随机生成流程，方便测试、调试和场景定制。

---

## 1. 改动目标

正常创建玩家时，八围属性和技能等级完全由名字的 RC4 推导决定，无法精确控制。在以下场景中需要可预测的固定数值：

- **测试对齐**：验证特定属性/技能组合的行为
- **场景定制**：预设 BOSS / NPC 的精确数值
- **快速调试**：跳过名字筛选直接验证技能逻辑

本次新增的 overlay 系统通过名字后缀的 `diy[...]{...}` 或 `ol:{...}` 格式提供精确覆盖，同时保持正常通路零开销。

---

## 2. 变更清单

### 2.1 新增 `PlayerOverlay` 类型 (`player/overlay.rs`)

| 字段     | 类型                           | 说明                                                       |
| -------- | ------------------------------ | ---------------------------------------------------------- |
| `attrs`  | `Option<[i32; 8]>`             | 八围覆盖值（`[atk, def, spd, agi, mag, res, wis, maxhp]`） |
| `skills` | `Option<HashMap<String, u32>>` | 技能名 → 等级映射                                          |
| `weapon` | `Option<String>`               | 武器名覆盖                                                 |

支持两种解析格式：

| 格式                 | 示例                                                                   | 说明                          |
| -------------------- | ---------------------------------------------------------------------- | ----------------------------- |
| `diy[attrs]{skills}` | `diy[72,39,69,76,67,66,0,84]{"sklfire":5}`                             | 紧凑格式，属性值 -36 后取非负 |
| `ol:{...}`           | `ol:{"attrs":[1,2,3,4,5,6,7,8],"skills":{"fire":4},"weapon":"剁手刀"}` | JSON 对象格式，属性值原样使用 |

### 2.2 名字解析变更 (`player/impl_attr.rs`)

- `new_from_namerena_raw()` 支持在 `+` 后缀段中识别 overlay 格式
- 新增 `split_weapon_overlay()` 按 `+` 拆分段，分离武器名和 overlay
- 兼容旧版 `diy{xxxx}` 格式（无方括号的不解析为 overlay）

### 2.3 构造函数变更 (`player/impl_ctor.rs`)

- 新增 `new_and_init_with_overlay()` 构造函数，显式传入 overlay 数据
- `new_and_init()` 变为委托入口，内部调用 `new_and_init_with_overlay(..., None, ...)`
- overlay 有技能映射时，`skil_id` 使用固定布局（主动→被动）而非随机洗牌
- weapon 优先级：名字中的 `+weapon` 段 > overlay 中的 `weapon` 字段

### 2.4 属性构建变更 (`player/impl_attr.rs`)

- `build_inner()` 新增 `apply_overlay` 参数控制是否应用 overlay
- `build()` 传 `true`（应用 overlay），`build_for_clone()` 传 `false`（clone 不走 overlay）
- overlay 有八围时：跳过随机生成 + Boss 加成，直接使用覆盖值
- overlay 有技能时：跳过名字推导 + boost 流程，直接使用指定等级
- overlay 技能槽位使用固定顺序，保证 merge（吞噬）时槽位类型一致

### 2.5 新增技能工具函数 (`player/skill.rs`)

| 函数                                      | 说明                                                        |
| ----------------------------------------- | ----------------------------------------------------------- |
| `skill_name_to_id(name)`                  | 技能名 → ID 映射，兼容 `sklfire` / `FireSkill` / `fire`     |
| `diy_skill_order()`                       | 返回 overlay 模式固定槽位顺序（25 主动 → 10 被动 → 5 空槽） |
| `apply_diy_skill_levels(storage, levels)` | 将 overlay 技能等级写入 SkillStorage 并刷新 proc 缓存       |

新增常量：

| 常量                                 | 内容                                    |
| ------------------------------------ | --------------------------------------- |
| `DIY_ACTIVE_SKILL_IDS: [usize; 25]`  | 主动技能 ID 序列（Fire 0 ~ Shadow 24）  |
| `DIY_PASSIVE_SKILL_IDS: [usize; 10]` | 被动技能 ID 序列（Defend 25 ~ Hide 34） |

### 2.6 `Player` 结构体

- 新增 `pub overlay: Option<Box<PlayerOverlay>>` 字段
- `Box` 包装，无 overlay 时仅占一个指针大小（8 bytes）

---

## 3. 兼容性

- **纯增量**，所有公开 API 签名保持不变
- `new_and_init()` / `build()` / `build_for_clone()` / `new_from_namerena_raw()` 行为完全兼容
- 旧的 `diy{xxxx}` 格式（无方括号）不被识别为 overlay，武器名原样保留
- `overlay` 字段默认 `None`，现有代码无需改动
- `0.3.1` → `0.3.2` 为 patch 级步进，下游无需改代码即可升级

---

## 4. 性能

按 `docs/perf/benchmark_tracking.md` 的固定口径补跑（`--release --features no_debug`）：

| 样本           | 单线程 `100k` | 多线程 `100k` | 单线程 `1M` | 多线程 `1M` |
| -------------- | ------------: | ------------: | ----------: | ----------: |
| `aaa` vs `bbb` |      `1.901s` |      `0.301s` |   `19.333s` |    `2.800s` |
| `复杂样本`     |      `2.951s` |      `0.396s` |   `29.437s` |    `3.885s` |

和 `0.3.1` 基线对比（1M 口径）：

| 样本           | 配置   |   `0.3.2` |   `0.3.1` |    差异 |
| -------------- | ------ | --------: | --------: | ------: |
| `aaa` vs `bbb` | 单线程 | `19.333s` | `19.339s` |   ~持平 |
| `aaa` vs `bbb` | 多线程 |  `2.800s` |  `2.884s` | 快 2.9% |
| `复杂样本`     | 单线程 | `29.437s` | `30.181s` | 快 2.5% |
| `复杂样本`     | 多线程 |  `3.885s` |  `3.811s` | 慢 1.9% |

四项 1M 偏差均在 ±3% 以内，属于同机重跑的正常波动范围。**非 DIY 通路无性能回退** —— 所有新增判断都是对 `Option<PlayerOverlay>`（`None`）的指针判空，分支预测器稳定命中。

## 5. 使用示例

```rust
use tswn_core::player::Player;
use std::sync::Arc;
use tswn_core::engine::storage::Storage;

let storage = Storage::new_arc();

// 紧凑格式：八围属性值和技能等级
let mut player = Player::new_from_namerena_raw(
    r#"mario+diy[72,39,69,76,67,66,0,84]{"sklfire":5,"reflect":2}"#.to_string(),
    storage.clone(),
).unwrap();
player.build();
assert_eq!(player.attr, [36, 3, 33, 40, 31, 30, 0, 48]);
assert_eq!(player.skills.skill_by_id(0).level(), 5);  // fire skill

// JSON 对象格式：带武器名覆盖
let player = Player::new_from_namerena_raw(
    r#"luigi+ol:{"attrs":[1,2,3,4,5,6,7,8],"skills":{"fire":4},"weapon":"剁手刀"}"#.to_string(),
    storage.clone(),
).unwrap();
assert_eq!(player.weapon, Some("剁手刀".to_string()));
```
