# tswn_core 0.3.1 更新说明（ChargeSkill / AssassinateSkill 查询接口）

> `0.3.1` 在 `SkillTrait` 上新增两个查询方法，供 WASM 前端展示蓄力 step 和潜行目标时使用。

---

## 1. 改动目标

WASM 端 `collect_states()` 里的状态标签展示需要显示蓄力的当前 step 数值（类似狂暴/魅惑的剩余回合数）和潜行的锁定目标 ID（类似守护的 `守护 #N 中`），但 `SkillTrait` 没有暴露这两个字段的读接口。

本次在 core 层补齐接口，WASM 层直接调用使用。

---

## 2. 变更清单

### 2.1 `SkillTrait` 新增方法

| 方法 | 返回值 | 默认值 | 实现者 |
|------|--------|--------|--------|
| `charge_step()` | `i32` | `0` | `ChargeSkill` → `self.step` |
| `assassinate_target()` | `Option<PlrId>` | `None` | `AssassinateSkill` → `self.target` |

### 2.2 `Skill` 委托

- `charge_step()` 委托 `self.skill_type.charge_step()`
- `assassinate_target()` 委托 `self.skill_type.assassinate_target()`

### 2.3 技能实现

| 文件 | 变更 |
|------|------|
| `act/charge.rs` | `ChargeSkill` 实现 `charge_step()` 返回 `self.step` |
| `act/assassinate.rs` | `AssassinateSkill` 实现 `assassinate_target()` 返回 `self.target` |

---

## 3. 兼容性

- 两个新方法都有默认实现，**不破坏**现有任何 `SkillTrait` 实现者的编译。
- `0.3.0` → `0.3.1` 为 patch 级步进，下游无需改代码即可升级。
