# tswn_core 0.3.0 更新说明（Breaking Change — 玩家属性 mp → magic_point 重命名）

> `0.3.0` 是一次**不向后兼容**的版本步进，核心变动是将 `PlayerStatus` 中的 `mp` 字段重命名为 `magic_point`，并推动所有调用方使用新的正式方法名。
>
> 这是自 0.2 系列以来的首次 major 级变更（pre-1.0 语义下 minor 步进），主要为了清理历史遗留的缩写命名，使 API 更自文档化。

---

## 1. 改动动机

`mp` 是早期从 JS/Dart 移植时保留的缩写命名。随着项目规模扩大，缩写在代码可读性和代码补全上都不如全称 `magic_point` 直观。

本次改动主要是 **API 层面的收束**——`PlayerStatus` 中已经存在 `magic_point` 字段与 `mp` 字段并存（同一含义），`0.2.22` 的 `impl_ctor.rs` 中已经通过 `magic_point()` / `set_magic_point()` 提供了正式方法，但 `mp()` / `set_mp()` 仍作为废弃 shim 保留。

`0.3.0` 将剩余的 `mp` 引用全部替换为 `magic_point`，废弃方法标上 `#[deprecated]` 但不立即移除。

---

## 2. 变更清单

### 2.1 结构体字段

| 文件                         | 变更                           |
| ---------------------------- | ------------------------------ | ---------------- | --- |
| `status.rs` — `PlayerStatus` | `mp: i32` → `magic_point: i32` |
| `status.rs` — `Default`      | `mp: 0` → `magic_point: 0`     |
| `status.rs` — `Display`      | 输出标签 `mp                   | {}`→`magic_point | {}` |
| `status.rs` — 文档注释       | 字段表格、示例输出同步更新     |

### 2.2 废弃方法（保留，标 `#[deprecated]`）

```rust
#[deprecated(note = "请使用 magic_point()")]
pub fn mp(&self) -> i32 { self.status.magic_point }

#[deprecated(note = "请使用 set_magic_point()")]
pub fn set_mp(&mut self, val: i32) { self.status.magic_point = val; }
```

### 2.3 技能实现中的调用更新

| 文件          | 变更                                                                                |
| ------------- | ----------------------------------------------------------------------------------- |
| `charge.rs`   | `owner.set_mp(owner.mp() + 32)` → `owner.set_magic_point(owner.magic_point() + 32)` |
| `disperse.rs` | 3 处 `target.set_mp(...)` → `target.set_magic_point(...)`                           |
| `merge.rs`    | 4 处 `.mp()` → `.magic_point()`，2 处 `.set_mp(` → `.set_magic_point(`              |

### 2.4 运行时代码

| 文件              | 变更                                                                                       |
| ----------------- | ------------------------------------------------------------------------------------------ |
| `impl_runtime.rs` | 4 处调试日志 `self.status.mp` → `self.status.magic_point`；格式标签 `mp=` → `magic_point=` |
| `engine_core.rs`  | 修复误导标签：`mp=` 原实际输出 `move_point`，已改为 `mv=`                                  |

## 3. 下游 crate 同步变更

| Crate                         | 变更                                                                           |
| ----------------------------- | ------------------------------------------------------------------------------ |

## 4. 迁移指南

### Rust 内部代码

```diff
- player.status.mp
+ player.status.magic_point

- player.mp()
+ player.magic_point()

- player.set_mp(val)
+ player.set_magic_point(val)
```

### C API

```diff
- snapshot.mp
+ snapshot.magic_point
```

### Python

```diff
- player.mp
+ player.magic_point
```

### WASM / JS

```diff
- state.mp
+ state.magic_point
```
