# tswn_core 0.2.22 更新说明（ProtectSkill 查询接口 + 状态标签展示增强）

> `0.2.22` 主要在 WASM 前端展示层做了大量增强，core 层只新增了一个小接口。
>
> 核心工作是让 show 页面的左侧角色面板能展示更详细的实时状态信息——每个状态标签带上具体数值（中毒层数、铁壁/疾走加成值、诅咒倍率、剩余回合数等），并让"守护"这个技能在面板上双向显示（被守护者显示"被 #id 守护"，守护者显示"守护 #id 中"）。

---

## 1. 改动目标

上一版（0.2.21）建立了状态标签的基础框架——让 WASM 层能收集每个玩家的实时状态并渲染成彩色 pill。

这版在此基础上做精细化：

1. **状态数值展示**：不只是显示"中毒"，而是显示"中毒 3层"；不只是显示"铁壁"，而是显示"铁壁 +5"。
2. **守护技能双向展示**：区分"被保护者"和"保护者"，分别显示不同的标签文案。
3. **ProtectSkill 查询接口**：新增 `protect_to_id()` 让外部能查到 ProtectSkill 当前保护的目标。

---

## 2. 主要改动

### 2.1 `SkillTrait::protect_to_id()` — ProtectSkill 目标查询

涉及文件：

- `crates/tswn_core/src/player/skill.rs`
- `crates/tswn_core/src/player/skill/skl/protect.rs`

**动机**：WASM 层需要知道"哪个角色正在保护谁"，以便在保护者的卡片上显示 `守护 #id 中`。

已有的 `clear_protect_to()` 是写入/清除操作，但没有对应的只读查询。新增的 `protect_to_id()` 填补了这个缺口：

```rust
// SkillTrait
fn protect_to_id(&self) -> Option<PlrId> { None }

// Skill（委托）
pub fn protect_to_id(&self) -> Option<PlrId> { self.skill_type.protect_to_id() }

// ProtectSkill 覆写
fn protect_to_id(&self) -> Option<PlrId> { self.protect_to }
```

默认返回 `None`，只有 `ProtectSkill` 覆写返回实际的 `protect_to` 字段。

这条接口的定位与 `clear_protect_to()` 对称——一个用于写入，一个用于读取，都不涉及战斗逻辑修改。

---

### 2.2 WASM 层状态标签数值化

涉及文件：

- `crates/tswn_wasm/src/fight.rs`

所有有数值的状态现在都带上具体数据：

| 状态 | 显示格式 | 数据来源 |
|------|---------|---------|
| 中毒 | `中毒 N层` | `PoisonState.count` — 中毒叠层数 |
| 铁壁 | `铁壁 +N` | `IronState.protect` — 防御加成值 |
| 疾走 | `疾走 +N` | `HasteState.faster` — 速度加成值 |
| 诅咒 | `诅咒 xN` | `CurseState.multiply` — 伤害倍率 |
| 狂暴 | `狂暴 (N)` | `BerserkState.step` — 剩余回合数 |
| 魅惑 | `魅惑 (N)` | `CharmState.step` — 剩余回合数 |
| 迟缓 | `迟缓 (N)` | `SlowState.step` — 剩余回合数 |
| 被守护 | `被 #N 守护` | `ProtectState.protect_from[0].owner` — 守护者 ID |
| 守护者 | `守护 #N 中` | `ProtectSkill.protect_to` — 被保护目标 ID |

数值为 0 时省略后缀（如 `铁壁` 不加 `+0`），保持标签简洁。

---

### 2.3 WASM 层守护标签双向展示

原来只有单向的"被保护者"检测（`ProtectState.is_some()`），现在拆分为两条独立的检测路径：

1. **被保护者**（身上有 `ProtectState`）：从 `protect_from` 中取第一个守护者 ID，显示 `被 #id 守护`
2. **守护者**（有 `ProtectSkill` 且 `protect_to` 有值）：通过 `protect_to_id()` 获取目标 ID，显示 `守护 #id 中`

一个角色可以同时显示两个标签（既被别人保护，又保护别人）。

---

### 2.4 前端列布局调整 & 玩家 ID 显示

涉及文件：

- `examples/show-render.js`
- `examples/show.css`

**列布局从 5 列缩减为 4 列：**

| 列 | 表头 | 数据 |
|---|------|------|
| 1 | 角色 | 名字 + ` #playerId` + HP/MP 条 + 状态 pill |
| 2 | HP | `hp/maxHp` |
| 3 | 蓝量/体力 | `mp / movePoint%`（合并显示） |
| 4 | 状态 | 状态文字 |

- 名字列加宽：`118px → 150px`
- 体力改为百分比显示：`movePoint / 2048 * 100`
- 玩家 ID `#id` 以灰色小字显示在名字旁

---

## 4. 向下兼容性

本次更新**完全向后兼容**：

- `protect_to_id()` 是新增方法，已有技能无需修改
- 未修改任何已有方法的签名或行为
- WASM 层 `PlayerState.status_labels` 字段语义不变，只是字符串内容更丰富了

---

## 5. 建议验证

- `cargo clippy -p tswn_core -p tswn_wasm`（零警告）
- 打开 `show.html` 运行战斗，观察左侧面板：
  - 状态标签是否带数值（中毒层数、铁壁加成等）
  - 有 Protect 技能的角色是否显示 `守护 #id 中`
  - 被保护的角色是否显示 `被 #id 守护`
  - 空格键能否暂停/恢复
- 多队战斗测试，确认 `protect_to_id()` 在多守护者场景下正确返回
