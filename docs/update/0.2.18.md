# tswn_core 0.2.18 更新说明（新增 diff 对账输出，修正 slow/haste 同层顺序，并补齐 runner 回归样例）

> `0.2.18` 这一版主要围绕两件事收口：第一，把近期排查 runner diff / 胜率偏差时临时需要的输出方式沉淀成正式 CLI 能力，新增 `tswn-cli diff`，让普通对战输入可以直接产出适合逐行对账的文本；第二，把 `slow` 与 `haste` 的状态结算顺序继续对齐到 JS 的“同层优先级 + 注册顺序打平”语义，修复一组近期定位到的 2v2 / 3v3v3 分叉样例。
>
> 这一轮同时把 runner 大样本回归用例从 `large_56` 继续拆分整理到 `large_56_61` 与 `large_62_65`，并补上 4 个新 case，方便后续继续沿编号扩展 diff_case。
>
> 对应简版条目见：`crates\tswn_core\CHANGELOG.md`

---

## 1. 改动目标

这轮改动的出发点比较明确：

- 近期已经定位到几组 JS / Rust 在 runner diff 与胜率上的小幅偏差；
- 这些偏差本质上不是“引擎整体跑偏”，而是局部时序和状态结算顺序上的细小分叉；
- 继续排查时，单靠现有普通 fight 输出或 raw 聚合输出都不够直接，缺少一条能稳定复现并适合文本比对的 CLI 入口。

因此，`0.2.18` 主要处理三件事：

1. 新增 `tswn-cli diff`，把普通对战输入直接转成 runner diff 风格输出；
2. 修正 `slow` / `haste` 的同层顺序语义，使 Rust 继续对齐 JS；
3. 把近期确认过的问题样例补进 runner 回归集，并整理测试文件结构。

## 2. 主要改动

## 2.1 新增 `tswn-cli diff` 子命令

涉及文件：

- `crates\tswn_core\src\bin\tswn_cli.rs`
- `crates\tswn_core\src\bin\tswn_cli\args.rs`
- `crates\tswn_core\src\bin\tswn_cli\fight.rs`

这一版给 CLI 增加了新的：

- `tswn-cli diff`

它的输入仍然是普通对战 raw 文本，也支持和现有 `fight` / `raw` 一样的输入来源方式：

- `--raw/-r`
- `--file/-f`
- stdin

目标不是再新增一套战斗语义，而是把现有 runner 推进过程转成更适合逐行比对的输出格式。这样在手动复现 diff_case、和外部 runner 做文本对账、或者排查某个 seed 的局部行为差异时，不再需要额外写临时脚本拼输出。

这一条也同步补进了 CLI 顶部注释和示例，避免后续只看源码头部文档时漏掉这条新命令。

## 2.2 diff 输出逻辑按 runner 对账格式整理

涉及文件：

- `crates\tswn_core\src\bin\tswn_cli\fight.rs`

为了让 `diff` 输出更适合直接文本比较，这轮没有复用原本面向人类阅读的普通 fight 输出，而是单独补了两层整理逻辑。

### 第一层：按事件流重组每一行

新增了专门的 `collect_diff_lines(...)`：

- 继续沿用 `runner.main_round()` 推进对局；
- 以 `UpdateType::NextLine` 作为分隔点聚合当前行；
- 每行内部把多个 update 用 `, ` 拼起来；
- 最终输出的是稳定的逐行文本，而不是夹杂 banner 或额外总结信息的终端展示结果。

也就是说，`diff` 不是“换个文案显示 fight”，而是明确面向“逐行可比对”的输出目标做了一次单独收口。

### 第二层：对消息做对账友好的格式化

新增了 `fmt_update_diff(...)` 与 `normalize_diff_trace_line(...)`，主要处理几件事：

- 把 `[0]` / `[1]` / `[2]` 这类模板占位替换成实际角色名或参数；
- 当 update 没有显式 `param` 时，根据 score 或 target 名称补出更接近 runner diff 预期的第三参数；
- 去掉 `[s_counter]`、`[s_dmg160]`、`[s_dmg120]`、`[s_dmg0]` 这类只对内部调试有价值、但会干扰文本对账的标记；
- 去掉括号包裹和行首特殊空白，把最终文本收成更稳定的对账格式。

这样处理后，`tswn-cli diff` 输出更接近当前 runner diff 样例和外部比对脚本真正关心的文本层，不需要每次先手工清洗一遍日志再看差异。

### banner 分流也同步调整

`diff` 和 `fight --out-raw` / `raw` 一样，都属于“输出应该尽量纯净”的路径。

因此这轮顺手把 banner 分流条件也补齐了：

- `ParsedCommand::FightDiff` 默认不打印 CLI 启动 banner。

这样输出从第一行开始就是对局文本，不会把欢迎语或版本提示混进 diff 结果里。

## 2.3 slow / haste 改为同层优先级，由注册顺序打平

涉及文件：

- `crates\tswn_core\src\player\skill\act\slow.rs`

这轮真正影响战斗语义的一处改动，是把 `SlowState` 的 `update_state_priority()` 从 `110` 调整到 `100`，与 `HasteState` 保持同一优先级层。

关键点不只是“数值改了”，而是明确回到 JS 那边的语义：

- 速度类状态本身处于同层；
- 同层状态谁先参与属性结算，由注册顺序决定；
- Rust 这边继续依赖 `PlayerStateStore` 的 `order` 来打破平局。

这样可以同时覆盖两类此前容易分叉的情况：

- 先挂 `slow`，后挂 `haste`
- 先挂 `haste`，后挂 `slow`

如果把 `slow` 单独放到更高优先级，虽然能修掉其中一类 case，但会把另一类 case 再次打坏。这一版改成同层后，最终以注册顺序决定先后，才能和 JS 的真实行为一致。

从结果上看，这一改动主要修复的是：

- 某些 2v2 diff_case 里的行动顺序偏移；
- 一组 3v3v3 回归样例里的后续连锁分叉；
- 由速度状态链先后顺序漂移带出的局部胜率偏差。

## 2.4 runner 回归测试扩充到 case 65

涉及文件：

- `crates\tswn_core\src\engine\test\runner\mod.rs`
- `crates\tswn_core\src\engine\test\runner\large_56_61.rs`
- `crates\tswn_core\src\engine\test\runner\large_62_65.rs`
- `crates\tswn_core\src\engine\test\runner\large_56.rs`

测试侧这轮做了两件事。

### 1. 把原来的 `large_56.rs` 拆成两个编号段

原来的 `large_56.rs` 里已经累计了 `56` 到 `61` 的样例。继续往后塞会让文件越来越难读，也不利于后续快速按编号定位 case。

因此这轮改成：

- `large_56_61.rs`
- `large_62_65.rs`

并同步更新 `runner/mod.rs` 的模块声明。

### 2. 新增 case 62 至 65

新增的 4 个 case 来自近期对账中已经确认可稳定复现的样例，文件内直接标记为：

- `diff_case 04`
- `diff_case 07`
- `diff_case 09`
- `diff_case 11`

这些样例覆盖的重点包括：

- `slow` / `haste` 同层顺序引起的行动链差异；
- 幻影、潜行、铁壁、魅惑等技能/状态交错时的 runner diff 文本一致性；
- 近期已经观察到的 1v1 / 2v2 / 多人混合对局分叉点。

原有 case 56 至 61 的断言保持不变，因此这轮不是“替换旧样例”，而是在保留已知覆盖面的前提下继续把新问题样本固化进去。

## 3. 使用示例

示例命令：

```bash
tswn-cli diff -r "mario\nluigi\n\npeach\nbowser"
tswn-cli diff -f input.txt
Get-Content .\input.txt | cargo run -p tswn_core --bin tswn-cli -- diff
```

适用场景包括：

- 手工重放某个 diff_case，直接看逐行输出；
- 和外部脚本、旧 runner、历史样例做纯文本对账；
- 在修状态顺序、技能时序、日志拼装问题时，快速确认修改是否只影响目标窗口；
- 把近期定位到的 mismatch case 固化为测试前，先通过 CLI 做一次肉眼复核。
