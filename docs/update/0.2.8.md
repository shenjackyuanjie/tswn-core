# tswn_core 0.2.8 更新说明（分段 case miner、缓存可见性与 SBY 排查流程补强）

> `0.2.8` 主要不是战斗语义大改，而是把大样本 SBY 排查这套工作流补完整：能稳定切片、能看见缓存、能避免跨区间误判回归，也把 `tswn-cli`/`tswn_case_miner` 的易错命令写清楚。
>
> 对应简版条目见：`crates\tswn_core\CHANGELOG.md`

---

## 1. 改动目标

`0.2.7` 之后，剩余 failed case 已经进入“少量但难排查”的阶段。

这时最大的问题不再只是“哪里和 JS 不一样”，而是：

- 能不能稳定跳过前面已经修过的区段；
- 能不能知道这次到底有没有真的重新调用 bun；
- 能不能避免把不同测试区间的结果拿来直接比较；
- 直接重放 failed case 时，agent 会不会又写错 Cargo bin 名。

所以 `0.2.8` 的重点，是给后续 SBY 修复提供更稳的排查基础设施。

---

## 2. 主要改动

## 2.1 case miner 支持按模式稳定分段排查

涉及文件：

- `crates\tswn_core\src\bin\tswn_case_miner.rs`
- `track_case_miner.py`

### 本次改动

- 新增 `--case-offset-per-mode`；
- 语义不是“全局跳过前 N 个样本”，而是对每种模式分别跳过前 N 个唯一 case；
- 这样可以直接从 `1v1 / 2v2 / 3v3v3 / ffa_4/6/8` 的后续稳定区段开始继续检查，而不用反复跑前面已修好的部分。

### 效果

这让 SBY 排查能真正按区段推进：

- 前 2000 个 case 修完后，可以直接看第 2001 到 4000 段；
- 不会因为前面的样本已经通过，就被迫整轮重跑同一批“已知没问题”的输入。

---

## 2.2 TS 缓存命中与 bun 调用可见化

涉及文件：

- `crates\tswn_core\src\bin\tswn_case_miner.rs`
- `track_case_miner.py`
- `target\ts_diff_cases\summary.json`（输出格式）

### 本次改动

- case miner 显式统计：
  - `ts_cache_hits`
  - `ts_cache_misses`
  - `bun_invocations`
- 这些统计会写进 `summary.json`；
- tracker 的非 quiet 模式下也会更直接地显示 Rust 编译与 miner 阶段输出。

### 效果

以前排查时很容易分不清：

- “这次是不是命中了共享 trace 缓存”；
- “是不是其实根本没重新跑 TS”；
- “卡住到底是编译、生成 case，还是在对比输出”。

`0.2.8` 之后，这些信息都能直接看到。

---

## 2.3 比较范围变化时不再误判回归

涉及文件：

- `track_case_miner.py`

### 本次改动

- tracker 在比较“本轮 vs 上轮/存档点”时，会把以下配置一起纳入比较：
  - `case_offset_per_mode`
  - `max_cases_per_mode`
  - `modes`
  - `ffa_sizes`
- 如果测试区间不一致，不再直接输出“修复了/退步了”，而是先提示比较范围已经变化。

### 效果

这避免了一个很容易误导排查方向的问题：

- 明明只是从“前 2000 个 case”切到“后 2000 个 case”；
- 结果 tracker 却把这两批完全不同的 failed case 当成修复或回归。

`0.2.8` 之后，这类跨区段误判会被挡住。

---

## 2.4 文档补齐 `tswn-cli` / offset 用法

涉及文件：

- `sby_test.md`
- `rule.md`

### 本次改动

- 明确写出普通 CLI 的 Cargo bin 名是 `tswn-cli`，不是 `tswn_cli`；
- 补充 `--out-raw --file` 的 failed case 重放示例；
- 同步记录 `--case-offset-per-mode` 的 PowerShell / bash 用法；
- 明确 offset 是“每种模式独立生效”的稳定切片。

### 效果

这批文档修正主要是在降低 agent 的常见误操作：

- 不再反复踩 `no bin target named tswn_cli`；
- 不再把 offset 理解成一个全局总偏移；
- 失败 case 的最小重放命令也更直接了。

---

## 3. 影响 / 对齐点 / 兼容性

`0.2.8` 没有新的战斗语义 breaking change，更多是排查与验证基础设施增强：

- case miner 的统计输出更完整；
- tracker 的回归判断更稳；
- 文档对 `tswn-cli`、offset 与固定 SBY 流程的约束更明确。

如果你在做 SBY 对账，这版的价值会非常直接；
如果你只是调用 crate 的运行时接口，本版行为变化很小。

---

## 4. 验证

简版 changelog 中记录的验证方式包括：

- `cargo fmt --package tswn_core`
- `cargo test -p tswn_core --bin tswn_case_miner`

整理时也重点核对了：

- `case_offset_per_mode` 是否按每种模式独立生效；
- 缓存命中 / miss / bun 调用次数是否落盘到 `summary.json`；
- tracker 在比较区间变化时是否停止直接判定回归；
- 文档中的 `tswn-cli` 重放命令是否与实际 Cargo bin 名一致。
