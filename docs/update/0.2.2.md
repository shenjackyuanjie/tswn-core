# tswn_core 0.2.2 更新说明（版本标识与 CLI 对外文案整理）

> 这次不是战斗逻辑大改，而是一次面向发布体验的整理：把 `tswn_core` 的版本标识、crate 描述和 CLI 启动提示调整到更适合对外使用的状态。
>
> 对应简版条目见：`crates\tswn_core\CHANGELOG.md`

---

## 1. 改动目标

`0.2.1` 之后，核心诉求不是继续压性能或改战斗语义，而是先把“这个 crate 是什么”“CLI 启动时怎么和用户说”整理清楚。

这一版主要做三件事：

- 明确 `tswn_core` 的 crate 版本从 `0.2.1` 推进到 `0.2.2`；
- 把 crate 描述改成更直接、对外可理解的表述；
- 精简 CLI 启动 banner 和警告文案，让当前阶段定位更清晰。

---

## 2. 主要改动

## 2.1 crate 元数据更新

涉及文件：`crates\tswn_core\Cargo.toml`

本次直接更新了 crate 元信息：

- `version = "0.2.1"` → `version = "0.2.2"`
- `description = "甚至是被技术封锁的名竞版本"` → `description = "甚至是rust版名字竞技场"`

### 调整原因

此前的描述更像开发阶段的内部玩笑或临时占位，不适合作为 crate 对外暴露的正式描述。`0.2.2` 先把这层元信息收正，便于后续文档、发布说明和使用方理解。

### 效果

- `tswn_core` 的版本号与 changelog 保持一致；
- crate 描述更贴近实际用途；
- 后续继续补版本文档时，版本和定位信息更统一。

---

## 2.2 CLI 启动文案整理

涉及文件：`crates\tswn_core\src\bin\tswn_cli.rs`

CLI 启动时的欢迎语和警告信息做了小幅收口：

- 欢迎语去掉了更强个人口吻的补充文本；
- 阶段提示从 `PRE ALPHA` 调整为 `ALPHA`；
- “发现行为不一致”提示保留，同时补充了更明确的联系入口信息。

### 调整原因

这一版想先把启动信息从“开发时随手写的文本”收成“可长期保留的提示”。

它不改变任何对战行为，但会直接影响：

- 新用户第一次运行 CLI 时的观感；
- 外部测试者对当前稳定度的理解；
- 报 bug / 反馈时的入口可见性。

### 效果

- CLI 的第一屏输出更简洁；
- 当前阶段的定位更明确；
- 对外分发和截图记录时，信息更稳定、可复用。

---

## 2.3 细节文案与示例同步

涉及文件：`crates\tswn_core\src\bin\tswn_cli.rs`

除 banner 外，本次也顺手清理了 CLI 文件中的少量注释/示例细节，使版本描述、帮助语境和启动输出更一致。

这部分不属于功能性改动，但对后续继续补 CLI 文档、示例和更新说明是有帮助的：至少版本标识、工具语气和对外描述不再互相打架。

---

## 3. 影响 / 兼容性

这版**不涉及战斗语义变更**，也没有新增 Rust API 层面的 breaking change。

影响主要集中在：

- crate 元信息；
- CLI 启动显示内容；
- 对外发布时的版本观感与描述一致性。

换句话说，`0.2.2` 更像一次“发布包装层整理版”，不是运行时逻辑版。
