# tswn_core 0.3.9 更新说明

`0.3.9` 是一次以 **行为对齐修复** 和 **CLI 功能补全** 为主的版本。相比 `0.3.8`，这一版的改动没有刻意扩展新玩法，而是集中处理 `0.3.8` 大规模优化后暴露出来的引擎边界差异，同时补齐 `batch-rate`、`bench pair`、`to-diy` 等 CLI 路径中缺漏的功能和输出模式。

本轮修复覆盖了回合同步、状态评分、中毒解除、浮点边界、召唤物护佑候选和 DIY SlotBoost cap 共六个独立对齐项；CLI 方面，`batch-rate` 新增纯名单 / JSONL / 分数三种输出模式、胜率精度控制和名字去重，`to-diy` 新增输出模式，`bench pair` 作为全新子命令落地，`namer-pf` 的 inline overlay 解析也一并修复。

## 版本定位

- 当前版本：`tswn_core 0.3.9`
- 上一版本：`tswn_core 0.3.8`
- 重点方向：
  - 对齐 md5.js 参考实现的六个行为差异点；
  - CLI 输出模式、精度控制与去重补全；
  - `bench pair` 子命令落地；
  - WASM 0.2.8：tsify 强类型导出；
  - 差分检测工具链增强。

## 行为对齐修复

`0.3.9` 的核心工作是六项行为对齐修复，目标是消除 Rust 实现与 md5.js 参考实现之间的残留边界差异。这些差异在 `0.3.8` 优化之后通过 `find_bun_tswn_namer_pf_mismatches.py` 等差分工具定位并逐个修正。

### 回合同步边界对齐

修复复活、召唤、关联召唤物清理和地震终止的回合内执行顺序，使其更接近 md5.js 的回合推进语义。

此前的回合结构中，这几个操作在回合内的执行时机与 JS 参考实现存在微妙错位，导致部分涉及召唤物 / 复活的场景在 namer-pf 评分中出现 `pp` / `pd` / `qp` / `qd` 分叉。本版本对 `engine_core`、`tick`、`world_state` 和 `quake` 的回合边界做了精细调整：

- 复活在回合内的触发时机提前；
- 召唤物 / 关联召唤物清理与战斗状态刷新更好地对齐；
- 地震技能终止在回合边界上的表现更稳定。

涉及文件：`engine_core.rs`、`tick.rs`、`world_state.rs`、`quake.rs`。

### 玩家状态评分行为对齐

按 md5.js 对齐以下五个状态相关路径：

- **中毒挂载**：中毒状态的初始挂载方式调整为与 JS `PoisonState` 一致的路径；
- **负面状态清理**：`post_defend` 中的负面状态清理时机和范围更精确；
- **pre_defend 零伤害早停**：防守阶段如果伤害已被削减至零，提前终止后续伤害钩子，避免多余状态触发；
- **吸血浮点边界**：`absorb` 吸血量的浮点计算边界与 JS 对齐；
- **隐匿状态边界**：`hide` 隐匿状态的结束判定条件对齐。

这些修复整体上减少了 Rust 与 Bun 评分对比中的残留分叉。涉及 `impl_ctor.rs`、`impl_runtime.rs`、`absorb.rs`、`fire.rs`、`poison.rs`、`hide.rs`、`state.rs`。

### 解除中毒不刷新运行时属性

修复中毒解除时的运行时属性刷新行为。按 md5.js 的 `PoisonState.K()` 语义，中毒解除只移除状态和注销 `post_action`，**不触发 `updateStates`**。此前的实现误在解除中毒时触发了全局状态更新，导致疾走等 pending 状态在中毒解除时提前生效，造成 namer-pf 中 `qp` 多 1 的边界差异。

### 浮点常量边界对齐

修复 `clone` 和 `ice` 技能中浮点常量与 JS 实现的边界差异。两处技能计算路径中浮点运算的常量边界与 md5.js 存在微小偏移，在特定数值组合下导致技能效果分叉。本版本将常量边界与 JS 对齐，并补充了针对这些边界的回归测试。

### 死亡召唤物护佑候选对齐

修复护佑技能在计算候选目标时对已死亡召唤物的处理。按 md5.js 的语义，已死亡召唤物不应出现在护佑候选列表中。此前的实现未正确过滤死亡状态的召唤物，导致护佑目标候选偏差。修复后 `protect.rs` 中增加了死亡状态判断，`storage.rs` 中也相应调整了候选集的维护逻辑。

### DIY SlotBoost cap 修复

修复 DIY 构造路径中 `SlotBoost` 加成量未按当前技能等级上限封顶的问题。

普通构造路径中，`boost_level` 会通过 `min(name_base_val, skill.level())` 对加成量上限封顶，clone 技能在衰减后等级降低，有效加成也会随之减少。但步骤 B（DIY 路径）之前直接应用了完整的原始加成量，未执行 `boost.min(current)` 封顶，导致 DIY clone 在 owner 已衰减时获得比普通 clone 更高的 SlotBoost 加成。

本版本在 `impl_attr.rs` 的 DIY 加成路径中增加了 `boost.min(current)` 封顶逻辑，与普通构造行为保持一致。验证：223/223 测试通过，1v1 500/500，ffa_6 614/614。

## CLI 功能增强

### `bench pair` 子命令

新增 `tswn-cli bench pair` 子命令，用于计算玩家与队友的组合胜率评分。

`bench pair` 会先读取 `player-list`（每行一个名字），将非 DIY/OL 的名字自动转换为 `+ol` 格式，然后与 `teammate-list` 中的每个队友组成二人组。对每个二人组计算一次 batch rate，把最高的 `--head <N>` 个 batch rate 求和作为该选手的最终分数，方便评估选手与不同队友搭配时的最优组合表现。

用法：
```powershell
cargo run -p tswn_core --bin tswn-cli -- bench pair \
  -l targets.txt -p players.txt \
  --teammate-list teammates.txt --head 3
cargo run -p tswn_core --bin tswn-cli -- bench pair \
  -l targets.txt -p players.txt \
  --teammate-list teammates.txt --head 5 -o pair.txt --min-file 250
```

### `batch-rate` 输出模式

`batch-rate` 子命令新增三种输出模式，通过 `-o <file>` 指定：

- **`--pure`**：输出纯名字列表，每行一个名字，仅包含达到 `--min-file` 阈值的选手；
- **`--log`**：输出 JSONL 格式，每行包含选手名、各项评分和胜率等完整信息；
- **默认**：输出文本格式，包含名字和分数。

三种模式满足不同下游消费场景：`--pure` 适合直接作为下一轮名字池，`--log` 适合程序化分析，默认模式适合人工阅读。

### `batch-rate` 胜率精度控制

新增 `--wr-precision <N>` 选项，控制 batch-rate 输出中胜率的显示精度（小数位数）。默认精度保持向后兼容。

### `batch-rate` 名字去重

修复 `batch-rate` 输入中重复名字的处理。当 `player-list` 或 `target-list` 中出现重复名字时，不再导致重复计算或输出，内部自动去重并给出警告提示。

### `to-diy` 输出模式

`to-diy` 子命令新增 `--old` 标志，可输出旧版格式的 DIY 字符串，方便与历史数据对比和迁移。默认输出保持当前紧凑格式。

### `namer-pf` 内联 overlay 解析修复

修复 `namer-pf` 命令对输入中包含 `diy[...]` 或 `ol:{...}` 内联 overlay 格式的解析。此前在部分边界情况下 inline overlay 可能被错误切分或忽略，修复后 `namer-pf` 可以正确识别和构造带 overlay 的玩家。

## WASM

### wasm 0.2.8 发布

`tswn_wasm` 发布 0.2.8 版本，核心变化是引入 `tsify` 生成 TypeScript 领域类型，并将 wasm 导出接口从松散的 `JsValue` 改为强类型签名。这意味着通过 `wasm-bindgen` 生成的 `.d.ts` 文件中，参数和返回值现在具有明确的 TypeScript 类型，IDE 自动补全和类型检查得到显著改善。

### Replay Viewer UI 调整

调整了 WASM replay viewer 的控制面板布局和样式，使其在桌面和移动端浏览器上都更易操作。

## 工具链

### namer-pf 差分检测脚本

新增 `scripts/find_bun_tswn_namer_pf_mismatches.py`，用于批量对比 Bun (md5.js) 和 Rust (tswn-cli) 的 namer-pf 评分输出，定位分叉 case。脚本支持：

- 解析多种格式的 namer-pf 输出（新版和旧版）；
- 按 pp/pd/qp/qd 四项分别对比并统计差异；
- `missing` 处理：当 Bun 或 Rust 端缺失某 case 时自动标记；
- 结合 `scripts/bun_score_trace.js` 实现分段评分追踪，用于精确定位评分分叉发生在哪个评估环节。

### bun_score_trace.js

新增 `scripts/bun_score_trace.js`，在 Bun 环境下运行 md5.js 并按步骤输出评分中间结果，辅助定位与 Rust 侧的评分差异来源。

### diff 工具改进

`find_bun_tswn_namer_pf_mismatches.py` 持续增强：支持重测模式，在重测时将当前 tswn-cli 结果与历史 Bun 分数对比；增加分段输出辅助，减少大海捞针式排查。

## 行为与兼容性

### 对普通对战的影响

本版本以行为对齐修复为主，不引入新的玩法语义。修复项均以消除与 md5.js 的差异为目标，对已接近对齐的场景影响极小，对存在分叉的场景会使 Rust 结果更接近参考实现。

### 对 CLI 的影响

`batch-rate` 输出格式增加了 `--pure`、`--log` 和 `--wr-precision` 选项，默认行为向后兼容。`bench pair` 是全新子命令，不影响现有命令。`to-diy` 新增 `--old` 模式，默认输出不变。

### 对 WASM 的影响

wasm 0.2.8 的 tsify 强类型导出是 breaking change — 如果 TypeScript 调用方直接依赖 `wasm-bindgen` 的 `JsValue` 松散签名，需要适配新的强类型接口。但 `demo.js` / `show-render.js` / `show-replay.js` 等示例已同步更新。

## 总结

`0.3.9` 是 `0.3.8` 之后的「收口 + 对齐」版本：

- 六个独立行为对齐修复，大幅减少 Rust 与 JS 的残留差异；
- CLI 功能补全，`batch-rate` 输出模式 / 精度 / 去重完善，`bench pair` 落地；
- WASM 0.2.8 强类型化，提升前端开发体验；
- 差分检测工具链成熟，为后续持续对齐提供精确定位能力。

如果从 `0.3.8` 升级，最明显的改进是 namer-pf 评分与 JS 的对齐度提升，以及 CLI 工具的可用性增强。如果只使用普通对战，本版本也会因多项边界修复而更加稳定。
