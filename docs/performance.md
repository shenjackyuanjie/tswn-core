## 1. 口径

- 编译参数：`--release --features no_debug`

  - `aaa` vs `bbb`

## 5. 结论

基于这次补跑的数据，当前可以比较明确地说：

- `0.2.14` 在吸收额外两条热路径优化后，单线程两条样本继续变快，复杂样本 CLI 多线程也继续变快；
- 相比回迁前的 `0.2.13`，当前主线在单线程两条样本上大致快 `7.6%~7.9%`，在复杂样本 CLI 多线程上快 `10.0%`；
- 相比当前 `opencode_branch`，当前主线在单线程两条样本和复杂样本 CLI 多线程上仍然更快；
- `aaa` vs `bbb` 的 CLI 多线程 `1M` 口径这轮只比 `0.2.13` 略快 `0.3%`，并且比 `opencode_branch` 慢 `2.8%`，这个点还需要继续观察；
- 同时，`RunUpdates` 复用导致的 `CounterSkill` 分叉问题已经在主线修掉，性能收益不是以牺牲结果一致性换来的。

也就是说，`0.2.14` 现在可以视作新的主线性能基线：

- 单线程更快
- 复杂样本多线程也更快
- 简单样本多线程仍需继续观察
- 且可回归验证
