# Performance

本文用于长期记录 `tswn_core` 在 `tswn-cli bench win-rate` 路径上的性能结果，方便横向比较：

- Bun / JS 历史参考
- `0.2.12`
- `0.2.13`
- `0.2.14`
- `0.2.19`
- `tswn_core_opencode` 当前分支

当前这份记录重点对应：

- `0.2.19` 在一轮结果一致性 / SBY diff 收口后，补跑当前 `bench win-rate` 口径的数据
- 这轮结果可以直接和 `0.2.14`、`opencode_branch` 做横向比较，观察“正确性收口后是否出现性能回退”

---

## 1. 口径

- 编译参数：`--release --features no_debug`
- CLI：`tswn-cli bench win-rate ... --perf`
- 单线程：追加 `--single-thread`
- 多线程：直接使用 CLI 默认线程策略
- 样本：
  - `aaa` vs `bbb`
  - `喘际瞬爆@昀澤` vs `蕾蒂·怀特洛可-65HEZHB264LFPFQ@Squall`

说明：

- `Bun` 与 `0.2.12` 的数据来自 `docs\update\0.2.12.md` 中的同机历史记录；
- `0.2.13` 数据来自 detached worktree：`f9b0e3c`；
- `0.2.14` 数据来自当时主线；
- `0.2.19` 数据来自当前工作树（已步进版本号并完成本轮 parity 修复）；
- `opencode_branch` 数据来自 `D:\githubs\namer\tswn_core_opencode` 当前分支；
- `100k` 更适合做快速横向参考；
- `1M` 更适合做当前版本之间的稳定吞吐比较。

## 5. 结论

基于这次补跑的数据，当前可以比较明确地说：

- `0.2.19` 这一轮首先完成了结果一致性收口；以 `tests/sqp5900.txt` 的 `3v3v3 + 4000 case` 口径看，`diff_failures` 已经回到 `0`。
- 但从 `bench win-rate` 看，当前工作树相较 `0.2.14` 基线出现了稳定的性能回退：
  - 单线程两条样本慢约 `3.0%~6.1%`
  - 多线程两条样本慢约 `9.0%~12.3%`
- 单线程还没有完全退回到 `0.2.13` 水平：`aaa` 与复杂样本相较 `0.2.13` 仍分别快约 `2.0%`、`5.1%`；
- 但多线程已经明显落后于 `0.2.14`，其中 `aaa` `1M` 多线程与复杂样本 `1M` 多线程都值得优先排查；
- 因此当前更适合把 `0.2.14` 继续视作性能基线，把 `0.2.19` 视作“正确性收口完成、但性能需要继续追”的新状态。
